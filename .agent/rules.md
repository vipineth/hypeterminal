# Code Style Rules

- Do not add comments explaining obvious code sections (e.g., `// Search`, `// Footer`)
- Do not add section comments in interface/type definitions
- Import order: external libs, then internal with `@/` alias
- Format: single-line ternaries when short, multi-line when complex
- Prefet funciton defination over arrow function
- Always change the version of the storage when you make changes to the store in zustand
